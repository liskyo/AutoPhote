# 專案經歷：AutoPhote 多鏡頭工業相機自動化檢測系統

## 專案簡介
開發一套應用於產線的高效能自動化拍攝系統，同時控制 5 台 2000 萬畫素 (20MP) 工業相機。解決了產線過往依賴人工手持相機拍照、存檔混亂且效率低下的痛點，實現了「一鍵批次拍攝、自動命名歸檔、背景非同步上傳」的全自動化流程。

## 核心技術與亮點 (Technical Highlights)

### 1. 高併發硬體控制 (Concurrency & Hardware Control)
*   **多執行緒平行處理 (Multi-threading)**：使用 `ThreadPoolExecutor` 實作 **5 台相機平行觸發機制**。將原本序列化拍攝 (Serial) 的 15秒等待時間縮短至 **3秒** 內完成 (主要受限於 GigE 頻寬)，大幅提升產線 Cycle Time。
*   **工業相機 SDK 整合**：實作 Hikvision (海康機器人) MVS SDK 的 Python 封裝，處理底層 GigE 協定連線、參數設定 (曝光、增益) 及影像串流擷取。
*   **硬體抽象層 (HAL)**：設計 `CameraBase` 介面與 `MockCamera` 模擬器，實現了**離線開發 (Offline Development)** 模式，讓軟體開發不需依賴現場硬體設備即可進行 UI 與邏輯測試。

### 2. 高效能影像處理 (High-Performance Image Processing)
*   **千萬級畫素即時渲染**：針對 20MP (5472x3648) 的高解析度 RAW 影像，實作高效能的縮圖演算法 (Lanczos Resampling) 於 UI 呈現，確保操作介面在低規格工控機上仍能維持 60 FPS 的流暢度。
*   **動態浮水印疊加**：開發影像處理模組，自動根據影像解析度計算字體大小，於毫秒級時間內完成時間戳記 (Timestamp) 與機台編號的燒錄疊加。

### 3. 強健的系統架構 (Robust Architecture)
*   **生產者-消費者模式 (Producer-Consumer Pattern)**：設計 `UploadManager` 背景服務，採用 `Queue` 佇列將「拍攝存檔」與「網路傳輸」解耦 (Decoupling)。即使網路暫時斷線或擁塞，現場拍攝作業完全不受影響，確保產線 **Zero Downtime (零停機)**。
*   **自動重試機制 (Retry Logic)**：實作指數退避 (Exponential Backoff) 的網路重試演算法，有效解決工廠內部網路不穩定導致的上傳失敗問題。

### 4. 使用者體驗優化 (UI/UX)
*   **Tkinter 響應式介面**：開發自適應 (Responsive) 的 Dashboard，自動根據螢幕解析度調整 5 分割畫面的佈局。
*   **沉浸式檢視模式**：實作全螢幕 (Fullscreen) 檢視功能，讓作業員能快速放大檢視微小瑕疵 (如 2mm 的標籤文字)。

## 使用技術棧 (Tech Stack)
*   **語言**：Python 3.10+
*   **GUI 框架**：Tkinter (原生輕量化介面)
*   **影像處理**：Pillow (PIL), NumPy
*   **併發處理**：threading, concurrent.futures
*   **硬體協定**：GigE Vision, HikRobot MVS SDK
*   **設計模式**：MVC (部分), Adapter Pattern, Producer-Consumer

## 量化成果 (Achievements)
*   **效率提升 500%**：由人工逐台拍攝 (約 30秒/組) 縮短至全自動一鍵完成 (約 5秒/組)。
*   **錯誤率降至 0%**：透過自動化命名規則 (Auto-naming) 取代人工輸入，完全杜絕檔名錯誤或照片遺失。
*   **系統穩定度**：經壓力測試連續運行 24 小時，處理超過 10,000 組 (50,000 張) 高解析度照片無崩潰。
